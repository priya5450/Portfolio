AI Video Ad Generation Platform ● Live
Replaced a multi-person manual production process with a single automated pipeline — full video in 5–10 minutes, under ₹16/video, replacing Creatify (USD SaaS).
LLMsCLIPFFmpegRedisCeleryElevenLabsSarvamDittoNginxDockerRunpod
+
The engineering journey
  • Researched how to build a video generation pipeline from scratch — no blueprint existed. Designed the architecture before writing a line of code.
  • Manually, producing one ad video needed a person to analyse the website and write the script, a voiceover artist, a videographer for avatar-based shoots, and an editor for effects and subtitles — a process spanning days per video.
  • Built modular stages: LLM script generation → CLIP image selection → audio synthesis → FFmpeg assembly → subtitle injection. Each stage independently swappable, and the full pipeline now runs end-to-end in 5–10 minutes with no manual intervention.
  • Hit concurrency wall under multi-user load. Added Redis + Celery task queue — async job processing, no timeouts, parallel generation.
  • Set up Nginx + Supervisord on Linux GPU servers for production-grade process management and reliability.
  • Added an avatar-based generation mode using Ditto for lip-sync: pass an audio URL and an avatar image, and it returns a fully lip-synced video — enabling personalised, avatar-led ads without any shoot. Shipped alongside Sarvam for Indian-accent voices, both based on direct user demand.
  • Replaced Creatify (USD-billed) entirely. Cost dropped to under ₹16 per video, eliminating vendor lock-in.
What made it hard
  • Coordinating async video stages without state corruption under concurrent load.
  • Keeping deployed system live while iterating — model swaps, new templates, new voices all shipped without downtime.
02
Customer-Facing AI Chatbot ● Live
Evaluated LangChain → identified bottleneck → migrated to OpenAI Agents SDK. Eliminated cold-start delay entirely, moving from a multi-second wait to an instant response on every query. Security-hardened before launch, with zero incidents since.
OpenAI Agents SDKLangChainLangGraphRAGChromaDBFastAPIDocker
+
The engineering journey
  • Started with LangChain + LangGraph and open-source LLMs. Output quality was insufficient for a customer-facing product.
  • Diagnosed the failure: agent reasoning loop was the bottleneck, not the model itself. Migrated to OpenAI Agents SDK — the right fix, not just a workaround.
  • Chose RAG over fine-tuning because the knowledge base needed frequent updates without retraining cycles. Implemented ChromaDB with semantic chunking.
  • Initial deployment on Render caused severe cold start latency — the first query after idle time could take several seconds. Researched root cause (serverless spin-up), documented tradeoffs, migrated to persistent GPU cloud, and eliminated the delay entirely — every query now gets an immediate response.
  • Ran dedicated security pass before go-live: prompt injection, hijacking, jailbreak attempts. Patched system prompt vulnerabilities and added an input sanitisation layer — zero security incidents since launch.
What made it hard
  • Knowing when to switch frameworks — not panicking early, not persisting too long. Required deep diagnostic work.
  • Making security testing systematic, not ad hoc. Built a test suite for adversarial inputs, not just manual checks.

04
Intelligent Text Correction System ⬡ Production
Solved corrupted text in AI-generated images. 4-stage pipeline: YOLO → OCR → LaMa inpainting → re-render. Eliminated full image regeneration fallbacks.
YOLOOCRLaMaOpenCVPython
+
The engineering journey
  • Root cause first: generative models treat text as visual texture — they can't reliably render brand names or CTAs. Required a dedicated correction layer outside the generation model.
  • Designed a 4-stage pipeline: YOLO detects text regions → OCR extracts erroneous string → LaMa inpaints cleanly → controlled re-renderer places correct text with style-matched positioning.
  • Core challenge: invisible blending after inpainting. Early versions left visible patch boundaries. Resolved by expanding mask margins and applying OpenCV alpha compositing at feather edges.
  • Deployed to production. Eliminated the previous fallback of full image regeneration — directly reducing compute cost and pipeline latency for every affected asset.
What made it hard
  • The blending problem was subtle — the inpaint looks correct in isolation but fails against the surrounding image texture. Required systematic testing across many image types.
